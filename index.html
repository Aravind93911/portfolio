Project Case Study

SQL Injection Detection and Mitigation Using Machine Learning

TechBazaar is a secure e-commerce web application designed as a cybersecurity demonstration platform. Its core objective is real-time detection, blocking, and auditing of SQL Injection attacks using a hybrid architecture combining weighted regex detection, Levenshtein Edit Distance similarity scoring, confidence-based risk classification, and a Python machine learning pipeline built with TF-IDF and Random Forest.

Executive Summary

Problem SQL Injection remains one of the most dangerous web vulnerabilities, enabling authentication bypass, data extraction, destructive operations, and command execution.
Solution A hybrid SQLi detection engine combining 14 weighted regex patterns, fuzzy string similarity via Levenshtein distance, heuristics, and an ML pipeline using TF-IDF with Random Forest.
Architecture Five-layer structure: UI → SQLD detection engine → Netlify backend → PostgreSQL database → SOC-style admin monitoring and audit.
Impact Enables in-browser low-latency attack blocking, defense-in-depth, attack logging, and model observability through a professional dashboard.

Detection Logic

Regex Phase

Uses 14 patterns to detect UNION attacks, tautology payloads, piggybacked queries, schema probing, time-based blind SQLi, file access attempts, version probes, comment truncation, and command execution attempts.

Similarity Phase

Uses normalized Levenshtein similarity to compare user input against known malicious patterns. Variants can still be detected even if exact signatures are evaded.

Heuristic Phase

Additional signals like multiple apostrophes and semicolons contribute to the final confidence score.

Confidence & Risk

Inputs are classified into None, Low, Medium, High, and Critical. Values above 30% are blocked and logged.

Machine Learning Pipeline

Dataset: Labeled SQLi and normal inputs.

Feature Engineering: TF-IDF vectorization.

Model: Random Forest classifier with 100 estimators.

Evaluation: 96.8% accuracy, 97.2% precision, 96.4% recall, F1 score 0.95.

Inference: Persisted model and vectorizer loaded for safe prediction workflow.

Security Measures

Frontend SQLD Detection

Blocks suspicious inputs before API calls are made.

Parameterized Queries

Uses PostgreSQL placeholders so user values remain data, not SQL syntax.

XSS Prevention

HTML entity encoding protects rendered attack logs against stored XSS.

Key Attack Types Covered

Tautology UNION Piggyback Comment Truncation Blind SQLi Time-Based Blind Error-Based Command Execution

Conclusion

TechBazaar demonstrates a complete applied-security workflow: frontend threat interception, machine learning-assisted classification, defense-in-depth through secure backend practices, and SOC-level observability. It combines cybersecurity operations, application security, and ML-driven detection into a practical real-world portfolio project.
